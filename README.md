# 🤖 CoinsBot (GainzAlgo Signal Edition)

CoinsBot is an advanced Telegram bot for monitoring cryptocurrency pairs from the **Coins.ph Pro API**. It uses a custom **GainzAlgo** indicator to generate specific BUY and SELL signals, complete with Take Profit (TP) and Stop Loss (SL) levels. The bot sends alerts only when a new signal appears and can generate detailed candlestick charts on demand.

## ✨ Features

- **Signal-Based Alerts**: Monitors the market hourly and sends alerts *only* when a new BUY or SELL signal is generated by the GainzAlgo logic.
- **Advanced Candlestick Charts**: Generates beautiful candlestick charts with BUY/SELL signals, TP, and SL levels plotted directly on the price action, mimicking professional trading terminals.
- **Automated TP/SL Calculation**: Automatically calculates suggested Take Profit and Stop Loss levels for every signal using the Average True Range (ATR).
- **Dual Price Display**: Shows current prices in both **PHP** and **USDT**.
- **On-Demand Analysis**: Instantly generate a full market report and signal chart for any supported coin with the `/price` command.
- **Health-Check Web Server**: Includes an Express server for uptime monitoring on deployment platforms like Render or Railway.
- **Resilient Polling**: Features robust error handling and auto-restart logic for stable, long-term operation.

## ⚙️ Requirements

Before you begin, make sure you have:

- **Node.js 18+** recommended
- A **Telegram bot token** from [@BotFather](https://t.me/BotFather)
- Your Telegram **chat ID**
- A valid list of tradable symbols supported by **Coins.ph Pro**

## 🔐 Environment Variables

Create a `.env` file in the project root:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_chat_id
COINS=BTCPHP,ETHPHP,SOLPHP,XRPPHP
PORT=3000
```

### Variable Notes

- `TELEGRAM_BOT_TOKEN` – Your Telegram bot token
- `TELEGRAM_CHAT_ID` – Default chat ID where scheduled alerts are sent
- `COINS` – Comma-separated list of symbols to monitor
- `PORT` – Port used by the Express health-check server

### Supported Input Formats

The bot normalizes user input for convenience:

- `BTC` → `BTCPHP`
- `BTCUSD` → `BTCUSDT`
- `BTCUSDT` → `BTCUSDT`
- `BTCPHP` → `BTCPHP`

> Scheduled monitoring uses the symbols from `COINS` exactly as configured.

## 📦 Installation


## ▶️ Running the Bot

```bash
npm start
```

When the bot starts, it will:

1. Load environment variables
2. Start the Express server
3. Initialize Telegram polling
4. Schedule an hourly check for new trading signals (`0 * * * *`)

## 🤖 Telegram Commands

| Command | Description |
|---|---|
| `/start` | Enable automated hourly alerts for new signals. |
| `/end` | Pause automated alerts. |
| `/restart` | Restart the Telegram polling connection manually. |
| `/now` | Trigger an immediate analysis for all configured coins and show the latest chart. |
| `/coins` | Show the list of coins being monitored for signals. |
| `/price [coin]` | Generate a detailed report and candlestick chart with GainzAlgo signals. |
| `/help` or `/commands` | Show the command list. |

### Example

```text
/price SOL
```

This will generate a full report for `SOLPHP`, including a candlestick chart showing the most recent BUY/SELL signals.

## 📊 Alert / Report Contents

Each alert or on-demand report includes:

- **Signal Banner**: A clear status like `[NEW BUY SIGNAL]` or `[HOLD]`.
- **Coin Label**: The symbol being analyzed (e.g., `◎ SOL ◎`).
- **Signal Details**: A summary of the latest signal, including suggested TP and SL levels.
- **Market Data**:
    - Current GainzAlgo momentum value.
    - Current price in PHP.
    - Current price in USDT.
    - 24-hour percentage change.
- **Candlestick Chart**: A high-quality image showing price action with signals plotted directly on it.

If chart generation fails, the bot gracefully falls back to a text-only message.

## ⏰ Scheduler

CoinsBot runs an automated check at the start of every hour for **new signals** on the last closed candle.

```cron
0 * * * *
```

## 🛡️ Reliability Features

The bot includes several protections for long-running operation:

- Graceful polling start/stop handling.
- Auto-restart with exponential backoff for recoverable polling errors (`409 Conflict`, `ECONNRESET`, etc.).
- Graceful shutdown on `SIGINT` / `SIGTERM`.
- Robust error handling for API failures or missing coin data.

## 📌 Notes

- An alert is only triggered when a *new signal* forms on the most recently closed 1-hour candle. The `/now` or `/price` commands will always show the latest market state, regardless of whether a new signal has formed.
- This project is intended as a market analysis tool, **not** an automated trading bot.
